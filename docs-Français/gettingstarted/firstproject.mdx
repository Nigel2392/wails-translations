# Création d'un projet

## Génération de projet

Maintenant que le CLI est installé, vous pouvez générer un nouveau projet en utilisant la commande `wails init`.

Choisissez votre framework préféré :

```mdx-code-block

import TabItem from "@theme/TabItem";

        {label: "Lit", value: "Lit"},
        {label: "Vanilla", value: "Vanilla"},
    ]}
>
<TabItem value="Svelte">
    Générez un projet <a href={"https://svelte.dev/"}>Svelte</a> en utilisant JavaScript avec :<br/><br/>

    wails init -n myproject -t svelte

Si vous préférez utiliser TypeScript :<br/>

    wails init -n myproject -t svelte-ts

</TabItem>
<TabItem value="React">
    Générez un projet <a href={"https://reactjs.org/"}>React</a> en utilisant JavaScript avec :<br/><br/>

    wails init -n myproject -t react

    wails init -n myproject -t react-ts

    wails init -n myproject -t lit-ts

</TabItem>
```

Il existe également des [modèles de communauté](../community/templates.mdx) disponibles qui offrent différentes fonctionnalités et frameworks.

Pour voir les autres options disponibles, vous pouvez exécuter `wails init -help`.
Plus de détails peuvent être trouvés dans la [Référence CLI](../reference/cli.mdx#init).

## Structure du projet

Les projets Wails ont la structure suivante :

``` 
.
├── build/
│ ├── appicon.png
│ ├── darwin/
│ └── windows/
├── frontend/
├── go.mod
├── go.sum
├── main.go
└── wails.json
```

### Présentation de la structure du projet

- `/main.go` - L'application principale
- `/frontend/` - Fichiers du projet frontend
- `/build/` - Répertoire de construction du projet
- `/build/appicon.png` - L'icône de l'application
- `/build/darwin/` - Fichiers spécifiques au Mac
- `/build/windows/` - Fichiers spécifiques à Windows
- `/wails.json` - La configuration du projet
- `/go.mod` - Fichier de module Go
- `/go.sum` - Fichier de vérification du module Go

Le répertoire `frontend` n'a rien de spécifique à Wails et peut être n'importe quel projet frontend de votre choix.

Le répertoire `build` est utilisé pendant le processus de construction. Ces fichiers peuvent être mis à jour pour personnaliser vos constructions. Si des fichiers sont supprimés du répertoire de construction, des versions par défaut seront régénérées.

Le nom de module par défaut dans `go.mod` est "changeme". Vous devriez le changer pour quelque chose de plus approprié.
